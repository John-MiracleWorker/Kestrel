# Runtime Flags

This file documents the platform-level flags that currently affect startup profiles and execution behavior.

| Flag                        | Current default                                                      | Class                      | Notes                                                                                                                                                                                                                                         |
| --------------------------- | -------------------------------------------------------------------- | -------------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| `KESTREL_FEATURE_MODE`      | `core`                                                               | Stable                     | Core, Ops, and Labs layering is implemented in Brain feature-mode policy. Defaults are now aligned to `core` across `.env.example`, Docker Compose, and the native-hybrid profile.                                                            |
| `KESTREL_RUNTIME_MODE`      | `docker` when unset; `native` in desktop profile                     | Stable                     | Controls Brain runtime routing between docker, native, and hybrid execution. Hybrid fallback to native is now surfaced explicitly in runtime results, task-event metadata, and audit traces instead of only appearing as an implicit warning. |
| `KESTREL_ENABLE_HOST_WRITE` | `false` in `.env.example` and native-hybrid; configurable in Compose | Experimental and high-risk | Enables host write tooling. Keep disabled unless policy is explicitly configured.                                                                                                                                                             |
| `USE_LANGGRAPH`             | `true` in Compose                                                    | Compatibility              | LangGraph is the preferred execution path. Legacy loop behavior still exists as fallback when disabled.                                                                                                                                       |
| `USE_LANGGRAPH_INTERRUPT`   | `true` when unset                                                    | Compatibility              | Controls LangGraph-native approval interrupts.                                                                                                                                                                                                |
| `ENABLE_OAUTH`              | `false`                                                              | Optional stable            | Turns on Gateway OAuth routes.                                                                                                                                                                                                                |
| `ENABLE_MAGIC_LINK`         | `false`                                                              | Optional stable            | Turns on passwordless email login routes.                                                                                                                                                                                                     |
| `TELEGRAM_MODE`             | `polling` when unset                                                 | Optional stable            | Selects Telegram webhook or polling mode when the adapter is enabled.                                                                                                                                                                         |

## Flags that still need cleanup later

- Provider-routing and model-selection defaults are spread across workspace settings and several environment variables.
- Native tooling policy flags need a clearer runtime profile surface in the UI and API.
